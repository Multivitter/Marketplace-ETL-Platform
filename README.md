Marketplace ETL Platform


Легкий, автономний ETL-пайплайн для збору фінансових та операційних даних з маркетплейсів (Amazon · Walmart · Shopify) + зовнішні джерела. Альтернатива SaaS-рішенням із повним контролем над пайплайном і нижчою вартістю володіння.




Мета проєкту

SaaS-сервіси для аналітики Amazon коштують дорого й тримають дані та логіку у себе. Мета — побудувати власну автономну альтернативу: повний контроль над пайплайном, дані у власній базі, мінімальна вартість володіння, можливість розширювати під будь-які внутрішні задачі без обмежень чужого продукту.

Що це робить

Повний цикл ETL (Extract → Transform → Load): тягне дані з API маркетплейсів за розкладом, нормалізує їх до єдиної схеми й завантажує у власну базу даних, поверх якої будуються дашборди й аналітика.


Архітектура

Джерела даних
  ├─ Amazon (SP-API)
  ├─ Walmart
  ├─ Shopify
  └─ зовнішні сигнали (напр. погода)
        │
        ▼
   Extract — авторизований збір за розкладом
        │
        ▼
   Transform — нормалізація до єдиної схеми, очистка, валідація
        │
        ▼
   Load — завантаження у PostgreSQL
        │
        ├──► BI-дашборди (аналітика, метрики)
        └──► внутрішні сервіси (алерти, прогнози, агенти)

Можливості


Мультиплатформенність — Amazon, Walmart, Shopify та зовнішні джерела в одному пайплайні з єдиною схемою даних.
Автономність — працює за розкладом без ручного втручання; дані у власній базі, не в чужому SaaS.
Безпечна авторизація — промисловий стандарт авторизації до API маркетплейсів (деталі реалізації приватні).
Надійність — обробка помилок API, повторні спроби, валідація даних на завантаженні.
Розширюваність — нові джерела й завантажувачі додаються модульно, поверх даних будуються дашборди, алерти, прогнози.
